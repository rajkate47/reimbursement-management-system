Problem statement

Manual reimbursements are slow, opaque, and easy to break.

Thresholds are hard to enforce

Finance teams need rules that shift review paths based on value, role, and policy without reworking the process every time.

Approvals stall between teams

Requests often disappear in email threads because there is no single queue for managers, finance, and directors.

Receipts and currencies add friction

Employees spend time typing line items while approvers struggle to compare costs in the company default currency.

Workflow engine

Sequential approvals with conditional decision logic.

The expense moves to the next approver only after the current approver approves or rejects it.

Core journey

  1. 1. Signup and company creation A company and admin user are created with the selected country currency.
  2. 2. Role assignment Admin creates employees and managers, then maps manager relationships.
  3. 3. Expense submission Employees submit amount, category, description, date, and receipt.
  4. 4. Ordered approvals Manager, finance, and director approve in the configured sequence.
  5. 5. Resolution and history Approved and rejected claims remain visible in the employee history.

Role permissions

Each role sees a different control surface.

Admin control

Configure the system

  • Create the company automatically on first signup.
  • Manage users, roles, and reporting relationships.
  • Define ordered approvers and conditional approval rules.
  • View all expenses and override decisions when needed.

Manager workspace

Review and escalate

  • See expenses waiting for approval in the default company currency.
  • Approve or reject with comments that remain visible in the audit trail.
  • View team expenses and escalate based on approval rules.
  • Act as the first approver when manager approval is required.

Employee view

Submit once, track clearly

  • Submit amount, category, description, date, and receipt details.
  • Enter expense values in a currency different from the company default.
  • Use OCR to auto-fill receipt fields and line items.
  • Track history and see approved or rejected requests in one place.

Key features

The product brief translated into interface moments.

01

OCR-powered receipt capture

Extract merchant name, line items, amount, date, and category directly from a scanned bill.

02

Country-aware currency defaults

Company currency is set during onboarding, while employee claims can still arrive in other currencies.

03

Multi-level approval sequences

Admins can define who approves first, second, and third so every handoff is traceable.

04

Conditional approval rules

Percentage thresholds, specific approvers like CFO, or hybrid rules can approve an expense automatically.
